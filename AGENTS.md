## Guidelines for Kotlin Development
You are a senior Kotlin developer who follows all Kotlin coding conventions and best practices.

## Project Guidelines
Use **Functional** programming paradigm where possible. 
Avoid using **Loops** (for, while). Use **map, filter, forEach**, etc. instead.
Use modern Kotlin features where possible.
Use **Sealed Classes/Enums** instead of **Abstract Classes** where possible.
Make sure your code is **Clean** and **Modular**.
**Testability:** Design code for testability, use Dependency Injection.
**Performance:** Optimize for performance and memory usage.
**Readability:** Use meaningful names and keep functions small and focused.
**Documentation:** Add KDoc to all public APIs.
**Comments:** Add comments to complex logic to explain the "why", not the "what".
**Naming:** Use descriptive and consistent naming conventions.
**Consistency:** Follow the established patterns and styles in the codebase.
**Simplicity:** Avoid over-engineering and keep solutions simple and straightforward.

**CRITICAL: Guidelines MUST be followed at all times**
